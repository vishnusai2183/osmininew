FIRST COME FIRST SERVE (FCFS)

The main principle of FCFS is, the process that requests the CPU first is allocated to the CPU first. Implementation of FCFS is managed using FIFO queue. When a process enters the ready queue, its PCB is linked onto the tail of the queue. When the CPU is free, it is allocated to the process at the head of the queue. The running process is then removed from the queue. This algorithm is nonpreemptive.

Example:
Let us consider 5 processes, P0, P1, P2, P3 and P4. The following are the respective arrival and burst times of the given processes.

The processes are sorted in the ascending order of their arrival times and then executed. Hence, the processes are executed in the order of P0, P1, P2, P3 and P4. We can calculate the values of completion time, waiting time and turnaround time of each process using the formulas given below.
Turnaround time = Completion time – Arrival time
Waiting time = Turnaround time – Burst time

By calculating the average of all the turnaround time values we can obtain the average turnaround time. Similarly, average waiting time can be computed.
Average turnaround time = Total turnaround time/number of processes
= ( 2+7+10+18+27 ) / 5
= 64/5
= 12.8

Average waiting time = Total waiting time/number of processes
= ( 0+1+6+9+15 ) / 5
= 31/5
= 6.2

Advantages:
  • The code for FCFS scheduling is simple to write and understand.

Disadvantages:
  • Average waiting time is long.
  • Due to the non-preemptive nature of the algorithm, the problem of starvation may occur.
  • Convoy effect occurs when other processes wait for the one big process to get off the CPU. This effect results in lower CPU and device utilization than might be possible if the shorter processes were allowed to go first.
  • FCFS is troublesome for time-sharing systems, where each process gets a share of the CPU at regular intervals.
