System Calls

Programmatic way in which a computer program requests a service from the kernel of the operating system on which it is executed. System calls provide an interface to the services made available by an Operating system.

WORKING OF SYSTEM CALL:
  1. Request of process from user space.
  2. Interrupt request for kernel.
  3. Control transfer to kernel.
  4. Kernel checks if operation can be done.
  5. Kernel performs operation.
  6. Kernel returns the output to the user process.
SYSTEM CALL CATEGORIES:
  1. Process control
  2. File Access, Directory and File system management
  3. Main memory management
  4. Device handling(I/O)
  5. Protection
  6. Networking

EXAMPLES OF SYSTEM CALLS IN THE ABOVE CATEGORIES AND THEIR FUNCTIONS:
  1. create()-
    A system call used to create a new empty file. The return type of this system call is integer. It has two parameters namely, filename and mode.
    int create(char *filename, mode_t mode);
  2. open() -
    A system call used to open a file for reading, writing or both. The return type is integer and it also has two parameters. Path denotes the path to file which we would use.
    int open(const char* path, int flags[, int mode ]);
  3. close() -
    A system call used to tell the operating system that you are done with a file descriptor(which is the parameter of close()) and close the file which is pointed by the given parameter(file descriptor).
    int close(int fd);
    Where fd file descriptor is a parameter and it returns 0 on success and -1 on error
  4. read() -
    From the file indicated by the file descriptor(integer parameter), the read() system call reads n bytes of input(integer parameter(int n)) into the memory area indicated by p(pointer(void* p) parameter).
    size_t read (int fd, void* p, int n);
  5. write() -
    A system call which writes n bytes(integer parameter) from p(pointer(void* p) parameter) to the file or socket associated with file descriptor(int parameter).
    size_t write (int fd, void *p, int n);
  6. lseek() -
    A system call that is used to change the location of the read/write pointer of a file descriptor
    off_t lseek(int fildes, off_t offset, int whence);
    Where fd or file descriptor , offset(offset to read/write) and whence(specifies offset relation and seek behavior) are the parameters.
  7. dup() -
    A system call creates a copy of a given file descriptor. This new descriptor does not behave like a copy, but like an alias of the old one.
    int dup(int oldfd);
    Where oldfd is the file descriptor to copy the parameter
  8. link() -
    A system call used to link a new file name to an existing file. However, it is impossible to say which file is the original one.
    int link(const char *oldpath, const char *newpath);
    Where oldpath(pointer to a string with old filename) and newpath(pointer to string with ne filename) are the parameters
  9. unlink() -
    A system call used to delete a file name from the filesystem. If that name was the last link to a file and no processes have the file open the file is deleted and the space it was using is made available for reuse.
    int unlink(const char *pathname);
    Where pathname(pointer to string with path name) is a parameter.
  10. access() -
    A system call which determines whether the calling process has access permission to a file. It can check any combination of read, write, and execute permission.
    int access(const char *pathname, int mode);
    Where pathname(file or device) and mode(permissions check to platform are the parameters. It returns 0 on success and -1 or error.
  11. chmod() -
    A system call used to change the access mode of the file. The name is an abbreviation of change mode.
    int access(const char *pathname, int mode);
    Where path(Pointer to string with name of file or device) and mode(new permissions made) are the parameters. It returns 0 on success , -1 on error
  12. chown() -
    A system call used to change the user or group ownership of a given file, directory, or symbolic link.
    int chown(const char *path, uid_t owner, gid_t group);
    Where path(pointer to string with name of file or device),owner(new owner of file or device) and group(new group of file or device) are the parameters. It returns 0 on success , -1 on error.
  13. umask() -
    A system call used to set the calling process's file mode creation mask (umask) to mask & 0777. The umask is used by open(2), mkdir(2), and other system calls.
    mode_t umask(mode_t mask);
    This system call always succeeds and the previous value of the mask is returned.
  14. ioctl() -
    A system call used for device-specific input/output operations and other operations which cannot be expressed by regular system calls.
    int ioctl(int d, int request, ...);
    Where d(open file descriptor the device file),request(request code) and â€¦( untyped pointer) are the parameters.
  15. execl() -
    A system call used to replace the current process image with a new process image specified by path. The new image is constructed from a regular, executable file called the new process image file.
    int execl(const char *path, const char *arg,...,NULL);
    For this system call the first and second arguments are the path of the executable file. Then the arguments we want to pass to the executable followed by NULL are taken. This system call runs the command and prints the output
  16. fork() -
    A system call used for creating a new process, called child process, which runs concurrently with the process that makes the fork() call(parent process). Child and parent both execute further, from the next statement of fork().
    pid_t fork(void);
  17. wait() -
    A system call that blocks the calling process until one of its child processes exits or a signal is received. After the child process terminates, the parent continues its execution after the wait system call instruction.
    A system call that blocks the calling process until one of its child processes exits or a signal is received. After the child process terminates, the parent continues its execution after the wait system call instruction.
    It takes one argument status and returns a process ID of dead children.
  18. exit() -
    A computer process terminates its execution by making an exit system call. More generally, an exit in a multithreading environment means that a thread of execution has stopped running.
    _exit()
  19. getuid() -
    A system call which returns the real user ID of the calling process.
    uid_t getuid( void );
  20. geteuid() -
    A system call which returns the real user ID of the calling process.
    uid_t getuid( void );
  21. getgid() -
    A system call that returns the real group ID of the current process.
    gid_t getgid( void );
  22. getegid() -
    A system call that returns the effective group ID of the current process.
    gid_t getegid( void );
  23. getpid() -
    A system call that returns the process ID of the calling process. This is often used by routines that generate unique temporary filenames.
    pid_t getpid( void );
  24. getppid() -
    A system call that returns the process ID of the parent of the calling process. It never throws any error therefore is always successful.
    pid_t getppid( void );
  25. signal() -
    A system call that installs a new signal handler for the signal with number num(integer parameter(int num) given to the system call signal()).
    sighandler_t signal(int num, sighandler_t handler);
    The signal handler is set to sighandler which may be a user specified function
  26. kill() -
    A system call that is used by the operating system to send a discontinuation signal to a process, urging it to exit. On the other hand, a kill system call does not always imply that the process is being terminated; it might have a variety of meanings.
    int kill(pid_t pid , int sig);
    This system call can be used used to send different types of signals.
    If pid>0 , then the signal sig is sent to the process with ID as specified by pid.
    If pid=0 , then sig signal is sent to every process in the process group of the calling process.
    If pid=-1 , then sig is sent to every process for which the calling process has permission to send signals.
    If pid <-1 , the sig is sent to every process in the process group of the calling process, whose ID is pid.
    If sig=0 , the no signal is sent.
  27. alarm() -
    A system call that causes the system to generate a SIGALRM signal for the process after the number of real-time seconds specified by seconds(parameter(unsigned int seconds) to alarm()) have elapsed.
    unsigned int alarm(unsigned int sec);
    This system call is used to send a signal to the calling process in sec seconds. If the sec is given as 0 then the pending alarm will be canceled. In any event previously set alarms are canceled.
  28. chdir() -
    A system call that changes the current working directory of the calling process to the directory specified in path(argument).
    int chdir(const char* path)
