MULTIPROGRAMMING WITH FIXED NUMBER OF TASKS (MFT):

MFT (Multiprogramming with a Fixed number of Tasks) is one of the old memory management techniques in which the memory is partitioned into fixed size partitions and each job is assigned to a partition. The memory assigned to a partition does not change.


The scheduler allocates memory to jobs based on the size of the job. Once loaded, the job can then compete for CPU time.


Best-Fit starts with the largest job and fits it in the smallest available space, until all possible allocations are made. It could also process the jobs in FIFO order, fitting each in the smallest space available. First-fit processes the jobs in FIFO order, fitting each in the first available space, until no more allocation is possible.

Advantages:
  • A fixed number of jobs at a time and this is known.
  • No frequent switching of boundary registers.
  • The strategy can be used along with swapping.

Disadvantages:
  • Job scheduling is an expensive overhead.
  • Fragmentation can be high.
    1. Internal fragmentation occurs when space is allocated but not used.
    2. External fragmentation occurs when space not allocated and not used.
  • There could be a job that is too large to run in the system.
