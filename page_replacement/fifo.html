FIFO PAGE REPLACEMENT

The main principle of FIFO, the process keeps track of all pages in the memory in a queue, the oldest page is in the front of the queue. When a page needs to be replaced page in the front of the queue is selected for removal. We can create a FIFO queue to hold all pages in memory. We replace the page at the head of the queue. When a page is brought into memory, we insert it at the tail of the queue.


This algorithm suffers with Belady’s Anomaly.

Page Fault – A page fault happens when a running program accesses a memory page that is mapped into the virtual address space but not loaded in physical memory.

Example:
Let us consider reference string 3, 2, 1, 0, 3, 2, 4, 3, 2, 1, 0, 4 with 3-page frames. Find number of page faults?

When we in the above example 1,2. If we increase the frames the page faults should decrease, but in 2nd example it was increased it is because of Belady’s Anomaly.

Belady’s anomaly:

Belady’s anomaly proves that it is possible to have more page faults when increasing the number of page frames while using the First in First Out (FIFO) page replacement algorithm. For example, if we consider reference string 3, 2, 1, 0, 3, 2, 4, 3, 2, 1, 0, 4 and 3 slots, we get 9 total page faults, but if we increase slots to 4, we get 10-page faults.

Advantages:
  • It is simple and easy to understand & implement.
  • It is efficiently used for small systems.
  • It does not cause more overheads.

Disadvantages:
  • The process effectiveness is low.
  • This algorithm suffers with Belady’s Anomaly.
  • Every frame needs to be taken account off.
  • It uses an additional data structure.
