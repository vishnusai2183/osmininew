OPTIMAL PAGE REPLACEMENT ALGORITHM :

In this algorithm, pages are replaced which would not be used for the longest duration of time in the future.

This algorithm that has the lowest page-fault rate of all algorithms and will never suffer from Belady’s anomaly.

Example:
Let us consider reference string 3, 2, 1, 0, 3, 2, 4, 3, 2, 1, 0, 4 with 3-page frames and 4-page frames. Find number of page faults?

Reference string:3 2 1 0 3 2 4 3 2 1 0 4



M-Miss
H-Hits
No.of page faults: 7
No.of Hits: 5

When we do same string with 4-frames



M-Miss
H-Hits
No.of page faults: 6
No.of Hits: 6

Compare to two examples, when we increase frames the page faults decreases.This algorithms doesn’t obey Belady’s Anomaly.Infact,this algorithm is best compare to other algorithms.

Advantages:
  • Complexity is less and easy to implement.
  • Assistance needed is low i.e Data Structure used are easy and light.

Disadvantages:
  • OPR (Optimal Page Replacement) is perfect, but not possible in practice as the operating system cannot know future requests.
  • Error handling is tough.
